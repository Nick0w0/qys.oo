# DEV_TASK_BREAKDOWN.md

配套文档：`FEATURE_REQUIREMENTS.md`、`DATA_MODEL_DRAFT.md`、`API_CHECKLIST.md`

本文把当前需求拆成可执行开发任务，便于后续按阶段实现、联调和验收。

## 一、已确认原则

- 子管理账号：一期一账号一学校。
- 技术设计：保留未来扩展为“一账号多学校”的能力。
- 权限原则：所有校级数据操作必须做服务端学校归属校验，不能只依赖前端隐藏。
- 安全原则：所有新增管理接口都要补输入校验、权限校验、操作日志。

## 二、任务拆分总览

### 阶段 A：学校基础数据

- 新增学校数据模型与后台维护能力。
- 明确学校唯一标识、名称、城市、经纬度、状态。
- 提供按地理位置推荐学校的基础查询能力。

### 阶段 B：后台管理员学校绑定

- 为子管理账号增加学校绑定能力。
- 总账号可新增、编辑、禁用、删除子管理账号。
- 子管理账号登录后仅可看到所属学校范围数据。

### 阶段 C：帖子学校归属与审核能力

- 帖子表增加学校归属字段。
- 补齐审核、置顶、删除所需字段与状态流转。
- 后台 discover 列表与操作接口按学校隔离。

### 阶段 D：小程序学校选择

- 首次进入请求定位并推荐学校。
- 支持手动搜索与切换学校。
- 当前学校写入用户态并影响首页/发布链路。

### 阶段 E：发布同步

- 帖子发布成功后按策略同步到指定渠道。
- 增加同步配置、日志、失败重试能力。

### 阶段 F：进群二维码弹窗

- 小程序进入时按策略弹出群二维码。
- 支持学校专属群码与平台默认群码。

### 阶段 G：首页广告活动

- 首页增加广告/活动运营位。
- 支持按学校、时间、状态、排序展示。

## 三、详细任务清单

### A1. 学校基础表与后台 CRUD

- 新增学校表。
- 后台新增学校管理菜单、控制器、模型、视图、JS。
- 支持学校启用/停用。
- 支持经纬度、城市、排序维护。

**完成标志**
- 后台可维护学校。
- 前后端可以通过学校 ID 关联业务数据。

### B1. 子管理账号学校绑定

- 基于现有 `application/admin/controller/auth/Admin.php` 扩展学校绑定字段/关系。
- 总账号创建子管理员时必须指定学校。
- 子管理员编辑时允许更换学校。
- 子管理员被禁用/删除后立即失效。

**安全要点**
- 服务端校验只有总账号才能改子管理员学校绑定。
- 防止普通子管理员通过直接请求修改自己所属学校。
- 写操作要有管理员日志。

### B2. 学校范围权限收口

- discover 后台列表默认附加学校过滤。
- discover 详情、审核、置顶、删除操作附加学校校验。
- 搜索条件、导出、批量操作都要附加学校条件。

**完成标志**
- 子管理员无法越权查看或操作其他学校帖子。

### C1. 帖子字段补齐

- 在 discover 主表增加 `school_id`。
- 增加审核状态字段。
- 增加置顶状态与置顶时间/排序字段。
- 评估是否保留现有 `statusdata`，避免把审核/隐藏/下架混在一个字段里。

**当前现状**
- 现有 discover 表有 `statusdata`、`province`、`city`、`latlng`。
- 现有 discover 表没有明确的 `school_id`、`audit_status`、`is_top` 字段。

### C2. 后台帖子审核 / 置顶 / 删除

- 在 discover 后台列表增加审核状态筛选。
- 增加审核通过/驳回动作。
- 增加置顶/取消置顶动作。
- 删除操作保留学校权限校验与日志。

### D1. 小程序学校选择链路

- 增加定位授权逻辑。
- 增加推荐学校接口。
- 增加学校搜索接口。
- 增加按城市筛选学校的能力。
- 页面在首次绑定前展示学校选择流程，并提供二次确认弹窗。
- 明确提示“选择后将无法自行修改”。
- 普通用户绑定学校后，不再提供自助切换入口。
- 发布帖子时强制带 `school_id`。
- 后台提供人工修改用户学校的能力，并记录日志。

### D2. 首页按学校展示

- 首页帖子列表默认按当前学校过滤。
- 无学校时提供兜底策略：手动选择、默认城市、平台推荐。

### E1. 发布同步配置与任务

- 定义同步目标配置表或配置项。
- 明确触发时机：发布后 / 审核后 / 置顶后。
- 实现同步日志记录。
- 实现失败重试。

**安全要点**
- 外部渠道凭据不得写死在仓库。
- 同步失败信息避免泄露敏感凭据。

### F1. 进群二维码配置与弹窗

- 新增群二维码配置数据。
- 增加学校维度配置。
- 前端增加弹窗组件和频率控制。
- 支持平台默认群码回退。

### G1. 首页广告活动位

- 新增广告活动数据模型。
- 后台支持上下架、排序、学校范围、跳转配置。
- 首页支持按学校加载有效广告。
- 点击进入对应目标页面。

## 四、建议开发顺序

### 第二优先级

- 首页按学校展示帖子
- 发布时带学校归属

### 第三优先级

- 进群二维码弹窗
- 首页广告活动位

### 第四优先级

- 发布同步
- 同步日志与重试

## 五、测试与验收清单

- 子管理员越权访问其他学校帖子应返回拒绝。
- 子管理员手动构造请求操作其他学校帖子应失败。
- 小程序拒绝定位时仍可手动选学校。
- 发布帖子未带学校时应明确报错。
- 首页广告只显示当前学校可见且在有效期内的内容。
- 二维码弹窗频率符合配置。
- 同步日志可追踪每次外发结果。

## 六、开发风险提示

- 当前 discover 表结构没有学校归属字段，后续所有列表和接口都需要补条件。
- 当前后台 discover CRUD 是 FastAdmin 生成式结构，权限收口不能只改页面，要同时改控制器查询与写接口。
- 当前存在 `application/api/controller/User.php` 与 `application/api/controller/discover/User.php` 两套用户接口，开发时要避免改错入口。
- 当前帖子表没有独立“审核状态/置顶状态”字段，建议不要继续复用 `statusdata` 承担全部语义。
